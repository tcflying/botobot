## 0. 当前定位

botobot 是一个**通用 agent 框架（harness）**；**coder bot 是它的第一个、当前唯一的 bot**——coding 不是终局，而是框架的第一性基本功。开发上以「自用、完全掌控、懂每一行」为优先，但框架身份不变。

```text
botobot = 通用 agent harness
  └─ coder bot      第一个、当前唯一的 bot
        文件理解 / 检索 / 安全执行 / 可审计编辑 / IDE 语义 / 多 session
     （researcher / operator / 多 bot 协作 = 后续，不预付抽象）
```

设计边界：

- `agent-*` 是通用 harness 层，不绑定 coding 场景。
- `coder bot` 是第一套角色配置：`webui-bin::profile::CoderBotProfile` 持有 role prompt、tool preset、policy preset 与 workspace 规则。
- 工具放在通用动作库里；是否暴露由 bot profile 决定。
- 「通用」是框架定位，但**不预付未实现的层**（world / multi-bot 等按真实需求再落，见 `todo.md §0`）。

## 2. 已完成的 agent 能力

- **OpenAI-compatible LLM**：支持 SSE 流式、tool-call、`reasoning_content` / `<think>` 推理分流、多模态 image parts。SSE 解析方言容错（§2.6 缺陷4，2026-06-25）：`usage` 接受 `input_tokens`/`output_tokens` 别名（Anthropic-compat 网关）、reasoning 接受 `thinking` 别名（serde alias，零风险，单测覆盖）。
- **本地 LLM 默认路径**：默认指向本地 unsloth/Qwen OpenAI-compatible endpoint，可由 `config.toml` 与 env 覆盖。
- **多工具调度**：默认并发执行，`ToolConcurrency::Exclusive` 可声明独占批次。
- **两级错误模型**：工具失败作为 observation 回喂；LLM/取消/超时/预算等 fatal 中止 turn。
- **413 请求过大恢复（§4.9 A2，2026-06-24）**：infer 返回 `Api{status:413}`（或 body 含 too large/payload）且本次发送含图片 → `strip_images`（图片 part 换占位文本，**只动发送副本不动 history**）后重试一次，仍失败才中止；无图片则照常中止。防 coder bot 截图喂入撑爆 body。
- **subagent 完成报告纪律（§4.9 A4，2026-06-24）**：`AgentTool` 派活尾部注入 `<system-reminder>`，要求子 agent 收尾输出 Result/Validation/Follow-up·blockers 结构化报告（纯 prompt、零架构）。
- **取消与超时**：`run_cancellable`、级联 `CancellationToken`、builder timeout。
- **exec policy / approval**：`Verdict` 支持 `Allow | Deny | Prompt`；driver 通过 `approval_request`/`approval_resolved` 事件暂停并等待响应，批准后继续，拒绝则作为工具失败回喂模型。**批准四档（§2.11 Stage 1，2026-06-23）**：`ApprovalDecision` = Once/Session/Always/Deny（`allows()`=非 Deny）；`ApprovalBroker` 持 per-session `session_allows` + `denied` 锁，发批准请求**前先查**（deny-lock→静默拒绝、allow-set→静默放行，fail-closed），响应后落表（Session/Always→放行集、Deny→拒绝锁）；`dedup_key`=工具名+原始 arguments（窄=安全，只对完全相同调用生效）。wire 向后兼容（`Op::Approval` 加可选 `decision`，缺省回退 `approved` 布尔）。**Always 持久化（Stage 2a，2026-06-23）**：`base-types::ApprovalStore` 端口 + `ApprovalBroker` 跨会话 `always` 集（启动载入、命中静默放行）+ webui-bin `FileApprovalStore` 落 `.bot/approvals.json`（加锁）经 builder 注入；end-to-end 测跨 agent 实例生效。**Stage 2b 前端 ✅（2026-06-24）**：approval 卡片渲染四档按钮（仅这次/本会话/永久/拒绝），点击经 WS 发 `{type:"approval", decision}`（保留 `approved` 布尔向后兼容），`approval_resolved` 回填卡片状态——§2.11 全链闭环。coder profile 默认用 **`agent-loop::RuleTableExecPolicy`（§4，借鉴 codex execpolicy）**：**沙箱模型（2026-06-25 用户拍板，取代旧白名单）**：classify 对 shell 命令判定——① **破坏性/整机灾难**（`rm -rf`/`dd`/`mkfs`/`shutdown`/`reboot`/`halt` + `find -delete` + 管道进 sh + fork bomb）→ **Deny**，**与路径无关**（workdir 内也拒，Q1）；② **触及 workdir 外**（绝对路径 `/…`/`C:\`/UNC、`..` 父级逃逸、无法分析的 `$()`/反引号）或 `find -exec` 跑任意子命令，且未显式信任 → **Prompt**（Q2）；③ **仅在 workdir 内**（相对路径、非破坏）→ **Allow**——含未知命令/串联/管道/重定向，**不再每条问**（消除 officecli/python 等 workdir 内命令逐条审批的痛点）。Forbidden 段扫描（串联/管道任意段）+ basename 归一（`/bin/rm -rf` 也拒）；越界检测按 token（`is_outside_path`）：绝对/盘符/UNC/`..` + **家目录引用**（`~`/`~/`、`$HOME`/`${HOME}`/`$env:USERPROFILE`/`%USERPROFILE%` 等无歧义家目录变量——堵 `cat ~/.ssh/id_rsa` 静默逃逸）。`allow` 表**默认含 `officecli`**（显式信任，见下止血），新语义 = 显式信任「即便越界也放行」的命令（经 `.bot/exec_policy.toml` 加；forbidden 恒胜 allow），按**段首**匹配（套 `$env:PATH=…; ## 3. 已完成的工具与资源层

- **统一 `read(url)`**：支持 `file://`、本地文件、`http://`、`https://`、`skill://`、`book://`、`artifact://`、`blob:sha256:`、`memory://`。
  - **单一归一化入口（§2.12，2026-06-23）**：`resource::normalize_file_path` 把 `file://` 后路径段/裸路径收敛成统一形态——折叠前导斜杠（`/Cargo.toml`→`Cargo.toml`、`/D:/x`→`D:/x`）、反斜杠转正斜杠、剥 `\\?\` 长路径前缀。**策略检查（`agent-loop::WorkdirPolicy::file_target`）与实际读取（`FileResource::resolve`）共用此函数**，消灭旧「两套规范化」错位（`file:///D:/…` 曾策略放行但读 os error 123，理论可构造「策略判安全/IO 读别处」）。workdir 内绝对路径 canonicalize 后放行；越界返回中文教学化纠错提示；read 工具描述/schema 明示「本地文件用相对路径，勿 file://」。
  - **provenance/credibility 头（§1.5 read 输出纪律）**：`skill://`/`book://` 读取返回自描述首行——`[SKILL procedural/gated] source=overlay|embedded`（skill_patch 演进过的附 `version=N`）、`[BOOK high-confidence] source=… section=… citation=book://…`（翻开即权威）。**`memory://` 改为按新近度/匹配分给三档（§1.8.7，2026-06-23）**：`[MEMORY] strong match — rely on it`（≥0.75）/ `partial match — sanity-check`（≥0.5）/ `faint match — verify before relying`（<0.5）/ 无命中 `no clear match`，每命中带 `(score=X)`。即「可信度判断点在召回时、随新近度衰减」，不再常驻里贴死 low-confidence。
- **read selector + hashline snapshot**：文件读取默认返回 `¶path#hash L<line> | <content>` 形式的稳定行锚；支持 `:N`、`:A-B` 与 `:raw` 选择器，后续 `edit_by_hashline` 可复用这些锚点做抗漂移编辑。
- **workdir Policy**：通用文件读受 `Context.workdir` 限制；托管 scheme 与托管写工具绕开普通文件闸。
- **Rust search / find**：`agent-act::search` 提供进程内 `search` 与 `find` 工具，使用 ignore-aware traversal 与 glob 过滤；`search` 输出结构化 `path/line/column/match/context`，并限制路径不逃逸 `ToolCtx.workdir`。
- **apply_patch**：`agent-act::patch` 提供 Write 级 `apply_patch` 工具，支持 Codex-style `*** Begin Patch` 格式的 Add/Delete/Update/Move、`@@` hunk、`dry_run` 预览、路径防逃逸与 hunk 错误定位；coder profile 默认暴露该工具。
- **edit_by_hashline**：`agent-act::edit` 提供 Write 级 `edit_by_hashline` 工具，输入 `path + hashline anchor + replacement`；执行前校验当前行 hash，漂移时拒绝并尽量给候选行，写入时生成 Codex-style patch 再走 `apply_patch`，支持 `dry_run`。
- **rename_file**：`agent-act::rename` 提供 Write 级 `rename_file` 工具，先尝试 rust-analyzer `workspace/willRenameFiles` 并安全应用工作区内文本 edits，再执行文件重命名；完成后 best-effort 发送 `workspace/didRenameFiles`，支持 `dry_run` / `overwrite` / `create_dirs`。
- **无通用代码 `write` 工具暴露**：coder preset 的常规代码写入入口是 `apply_patch` / `edit_by_hashline` / `rename_file`；托管写入仍由 memory/skill/artifact 等专用工具承担。
- **LSP 工具**：`agent-act::lsp` 提供 Read 级 `lsp` 工具；当前支持 `status`（Cargo.toml/cargo/rust-analyzer 可用性）、`diagnostics`（Rust `cargo check --message-format=json` 解析为结构化诊断），以及短生命周期 rust-analyzer LSP client 驱动的 `references`、`rename`、`code_actions`。`rename` 当前只返回 `WorkspaceEdit`，不直接改文件。

- **自定义 profile 文件入口**：默认仍为内置 coder profile；设置 `BOTOBOT_PROFILE=bots/<name>.toml` 或 `.md` 时会加载 profile 文件。TOML 支持 `id`、`display_name`、`system`、`replace_system`；Markdown 用一级标题派生显示名，正文作为 profile instructions 追加到内置 coder prompt。
